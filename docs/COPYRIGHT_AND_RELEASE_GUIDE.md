# Copyright, attribution, and archival release guide

Last reviewed: 2026-07-14

## Current rights structure

- Code: copyright © 2026 Advaith Praveen (APRK), licensed under MIT.
- Manuscript and original paper figures: copyright © 2026 Advaith Praveen (APRK), all rights reserved pending a later explicit publication agreement.
- Model checkpoints: distributed under the software license unless the eventual model host declares another compatible license.
- NASA PDS and third-party material: original terms apply; LunaSeis-1 claims no ownership.

MIT allows reuse, modification, and redistribution with the copyright and permission notice preserved. It supports an open repository but cannot prevent others from building on the code. It does not allow another person to erase the notice and truthfully claim original authorship.

## Recommended provenance sequence

1. Finalize the release candidate and independent review.
2. Create the public GitHub repository under the author's chosen account.
3. Create a signed or annotated immutable version tag.
4. Connect the repository to Zenodo and archive that exact release for a DOI.
5. Add the repository URL and DOI to `CITATION.cff`, the paper, model card, and README.
6. Preserve the release archive, SHA-256 manifest, submission receipt, and journal correspondence.
7. If desired, apply separately for Indian copyright registration for the manuscript and computer software.

The Copyright Office of India states that computer programs may be registered as literary works and describes the required filing materials. GitHub explains that open-source licenses authorize reuse under their terms, while an unlicensed work remains under default copyright. WIPO's Berne Convention summary describes automatic copyright protection among member states. Formal registration can strengthen evidence but cannot guarantee that plagiarism never occurs.
